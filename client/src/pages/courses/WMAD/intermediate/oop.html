1Ô∏è‚É£ The this Keyword in JavaScript

What is this?

this refers to the object that is executing the current function. Its value depends on how and where the function is called.

Example: Global vs. Object Context

console.log(this); // In browsers, this refers to the `window` object

const person = {
  name: "Alice",
  greet() {
    console.log(`Hello, my name is ${this.name}`);
  },
};

person.greet(); // "Hello, my name is Alice"

Common Issues with this

  • this inside an event listener may refer to the event target instead of the object.
  • In arrow functions, this is lexically bound (inherited from the surrounding scope).

2Ô∏è‚É£ Constructor Functions

            

                
What are Constructor Functions?

                
Special functions used to create multiple object instances.

                
                
Example:

                
function Car(brand, year) {
  this.brand = brand;
  this.year = year;
  this.getAge = function () {
    return new Date().getFullYear() - this.year;
  };
}

const car1 = new Car("Toyota", 2015);
console.log(car1.getAge()); // Outputs car's age

            

            

                
                
                
                

            

        


        

            
3Ô∏è‚É£ Prototypes and Prototype Inheritance

            

                
What is a Prototype?

                
JavaScript uses prototype-based inheritance, meaning objects inherit properties/methods from other objects.

                
                
Adding Methods to a Prototype:

                
function Animal(name) {
  this.name = name;
}

Animal.prototype.speak = function () {
  console.log(`${this.name} makes a noise.`);
};

const dog = new Animal("Dog");
dog.speak(); // "Dog makes a noise."

                
                
Why Use Prototypes?

                
Saves memory by sharing methods among multiple objects instead of defining them in each instance.

Why Use Classes?

5Ô∏è‚É£ Inheritance in Classes

            

                
What is Inheritance?

                
A mechanism to extend one class to another, allowing code reuse.

                
                
Example of Class Inheritance:

                
class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(`${this.name} makes a sound.`);
  }
}

class Dog extends Animal {
  speak() {
    console.log(`${this.name} barks.`);
  }
}

const dog = new Dog("Buddy");
dog.speak(); // "Buddy barks."

            

            

                
                
                

            

        


        

            
6Ô∏è‚É£ Getters and Setters in Classes

            

                
Why Use Getters & Setters?

                
Encapsulate logic for retrieving and modifying object properties.

                
                
Example:

                
class User {
  constructor(name) {
    this._name = name;
  }

  get name() {
    return this._name.toUpperCase();
  }

  set name(newName) {
    this._name = newName;
  }
}

const user = new User("alice");
console.log(user.name); // "ALICE"
user.name = "bob";
console.log(user.name); // "BOB"

            

            

                
                
                
                

            

        


        

            
7Ô∏è‚É£ Static Methods and Properties

            

                
What are Static Methods?

                
Static methods belong to the class itself, not instances.

                
                
Example:

                
class MathUtils {
  static add(a, b) {
    return a + b;
  }
}

console.log(MathUtils.add(5, 3)); // 8

            

            

                
                
                
                

            

        


        

            
8Ô∏è‚É£ The super Keyword in Class Inheritance

            

                
What is super?

                
It refers to the parent class and allows calling its constructor or methods.

                
                
Example:

                
class Vehicle {
  constructor(brand) {
    this.brand = brand;
  }

  start() {
    console.log(`${this.brand} is starting...`);
  }
}

class Car extends Vehicle {
  constructor(brand, model) {
    super(brand);
    this.model = model;
  }

  start() {
    super.start();
    console.log(`${this.model} is now running.`);
  }
}

const myCar = new Car("Toyota", "Camry");
myCar.start();

            

            

                
                

            

        


        

            
9Ô∏è‚É£ Object Composition (Alternative to Inheritance)

            

                
Why Use Composition?

                
Avoids deep inheritance trees, which can be hard to manage.

                
                
Example:

                
const canFly = {
  fly() {
    console.log(`${this.name} is flying!`);
  },
};

const canSwim = {
  swim() {
    console.log(`${this.name} is swimming!`);
  },
};

class Bird {
  constructor(name) {
    this.name = name;
  }
}

Object.assign(Bird.prototype, canFly);

const eagle = new Bird("Eagle");
eagle.fly(); // "Eagle is flying!"

            

            

                
                
                

            

        


        

            
üîü Private Fields in JavaScript Classes (ES2020+)

            

                
What are Private Fields?

                
Variables that cannot be accessed outside the class.

                
                
Example:

                
class BankAccount {
  #balance = 0; // Private property

  deposit(amount) {
    this.#balance += amount;
    console.log(`Balance: $${this.#balance}`);
  }
}

const account = new BankAccount();
account.deposit(100); // "Balance: $100"
// console.log(account.#balance); // Error: Private field
